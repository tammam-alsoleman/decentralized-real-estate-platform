# Blockchain Service Fabric Smoke Test

This smoke test invokes `SubmitContract` through the Blockchain Service
`FabricClientService`, then verifies the submitted record with
`GetContractByTxId`.

It is intended for local manual verification after the Fabric network and
chaincode are already running.

## Prerequisites

- Fabric CA and network containers are running.
- `realestatechannel` exists and the orderer/peers have joined it.
- `realestate-contract` chaincode is deployed on `realestatechannel`.
- PlatformMSP crypto material exists under:
  `infrastructure/fabric/organizations/peerOrganizations/platform.realestate.local`
- Blockchain Service dependencies are installed.

## Environment

From the repository root, copy the local Fabric example:

```bash
cp services/blockchain-service/.env.fabric-local.example services/blockchain-service/.env
```

The private key filename under `msp/keystore/` is generated by Fabric CA
enrollment. Before running the smoke test, update `FABRIC_PRIVATE_KEY_PATH` in
`services/blockchain-service/.env` to match the actual private key file present
under:

```text
../../infrastructure/fabric/organizations/peerOrganizations/platform.realestate.local/users/Admin@platform.realestate.local/msp/keystore/
```

Do not commit local `.env` files.

The local example assumes the Platform peer is reachable from the host at
`localhost:9051` and uses TLS server name override
`peer0.platform.realestate.local`.

## Run

```bash
cd services/blockchain-service
npm run smoke:fabric:submit
```

The npm script builds the service first, then runs the local smoke script
through `ts-node`:

```bash
ts-node -r tsconfig-paths/register src/scripts/fabric-submit-contract.smoke.ts
```

The smoke script is intentionally executed through `ts-node` because it is a
local manual verification utility and is not part of the Nest application
bundle emitted by `nest build`.

## Expected Result

The script should print a success summary showing:

- `SubmitContract` succeeded.
- `GetContractByTxId` returned the same `transactionId`.
- Contract status is `PENDING_EXTERNAL_APPROVALS`.

## Not Covered

This smoke test does not cover:

- RabbitMQ message consumption or publishing.
- Registry or Notary approvals.
- `ConfirmContract`.
- End-to-end Property Service integration.
